# Product: Levelup — 技能时间追踪

## Product Purpose

Levelup 是一个帮助用户追踪技能练习时间的工具。它让"时间投入"变得可见，帮助用户建立持续练习的习惯，通过数据可视化展示成长轨迹。

核心价值主张：让积累看得见，让成长有迹可循。

## Users

### 主要用户画像

**技能学习者**
- 年龄: 18-35岁
- 场景: 学习乐器、编程、绘画、语言等需要长期积累的技能
- 痛点: 缺乏持续动力，难以量化进步，容易半途而废
- 目标: 建立稳定的练习习惯，看到自己的成长曲线

**自学者**
- 年龄: 20-40岁
- 场景: 自学新技能、职业转型、个人成长
- 痛点: 没有外部监督，难以坚持，不知道自己投入了多少时间
- 目标: 用数据驱动学习，保持自律

**专业人士**
- 年龄: 25-45岁
- 场景: 精进专业技能、持续学习
- 痛点: 工作繁忙，需要高效利用碎片时间
- 目标: 追踪技能投入，优化时间分配

## Brand Identity

### 品牌个性

- **温暖但克制**: 像深夜工作室的台灯，专注而宁静
- **专业但不冰冷**: 数据驱动但有人文关怀
- **激励但不浮夸**: 真实的成长数据，而非虚假的鸡汤

### 品牌声音

- 简洁直接，不啰嗦
- 使用"你"而非"您"，保持亲近感
- 避免AI陈词滥调（Elevate, Seamless, Unleash）
- 数据说话，少用形容词

### 视觉隐喻

- 深夜观测站：追踪成长轨迹
- 积累的光点：每个练习时刻都是一个光点，汇聚成成长的轨迹
- 时间的重量：让抽象的时间变得具体可感

## Product Principles

### 1. 数据真实性

- 所有数据来自用户真实记录
- 不伪造统计数据或进度
- 空状态诚实展示，而非用假数据填充

### 2. 渐进式复杂度

- 新手可以快速上手记录时间
- 进阶用户可以设置目标、查看统计
- 高级用户可以自定义分类、导出数据

### 3. 专注核心功能

- 时间记录是核心，不做社交、不做排行榜
- 数据可视化服务于自我认知，而非攀比
- 保持工具的纯粹性

### 4. 离线优先

- 数据本地存储（localStorage）
- 可选的云端同步功能
- 无网络也能正常使用

## Feature Scope

### 核心功能（MVP）

1. **技能分类管理**
   - 树状分类结构
   - 自定义颜色标记
   - 分类目标设置

2. **时间记录**
   - 手动输入时长
   - 专注计时器
   - 日期选择

3. **数据看板**
   - 今日/本周/本月/总计统计
   - 连续打卡天数
   - 今日练习明细

4. **里程碑系统**
   - 自动检测时间阈值（1h, 10h, 50h, 100h...）
   - 成就展示

### 扩展功能

5. **目标追踪**
   - 为技能设置时间目标
   - 进度可视化

6. **数据统计**
   - 趋势图表
   - 分类分布

7. **数据同步**
   - GitHub Gist 同步
   - 多设备数据共享

## Anti-References

### 避免的设计模式

- **Duolingo式的游戏化**: 不做连续天数奖励、生命值、虚拟货币
- **Strava式的社交**: 不做排行榜、关注、动态流
- **Notion式的过度自由**: 不做可拖拽的模块化界面
- **传统仪表盘**: 不做密集的数据网格、多层嵌套卡片

### 避免的文案风格

- "你已经连续练习7天，太棒了！" → "连续 7 天"
- "距离目标还差2小时，加油！" → "还差 2 小时达成目标"
- "你的技能正在飞速提升！" → 不做虚假评价

## Success Metrics

### 用户价值指标

- 用户持续使用天数（留存率）
- 每日平均记录次数
- 目标完成率

### 产品健康指标

- 功能使用分布
- 错误率
- 性能指标

## Constraints

### 技术约束

- 纯前端应用，无后端依赖
- 使用 localStorage 持久化
- 响应式设计，支持移动端和桌面端

### 设计约束

- 深色主题（符合"深夜工作室"氛围）
- 使用 DESIGN.md 定义的设计系统
- 避免AI设计陈词滥调

### 业务约束

- 免费开源
- 无广告
- 无用户追踪
